**Task:** Generate or update comprehensive documentation (docstrings, comments, and README updates) for a specified code area.

**Input:** The user will specify the file, class, or function and the required documentation style (e.g., 'Python NumPy docstrings,' 'JSDoc style').

**Steps:**
1.  **Analyze Code:** Analyze the specified target (function, class, file) to understand its parameters, returns, side effects, and logic.
2.  **Implementation Plan:** Generate an Implementation Plan artifact outlining where docstrings will be inserted and any necessary explanations added to the README file.
3.  **Execute Documentation:** Insert or update docstrings for all functions and methods within the target area, adhering strictly to the specified style.
4.  **README Update:** If appropriate, generate a small summary for the component/module and append it to the relevant section of the main project README.md.
5.  **Verification:** Verify that the generated documentation is syntactically correct and covers all inputs/outputs.
6.  **Artifacts:** Generate a final artifact providing a markdown-formatted summary of the documentation created and the diff showing the changes to the README.

**Output Format:** Deliver the final result as a markdown artifact summarizing the generated documentation and confirming adherence to the requested style.
